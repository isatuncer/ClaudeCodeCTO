# CloaudeCodeCTO — Resume Point
> Last updated: 2026-04-12

## Project Goal

Scan public Claude Code GitHub repos (submodules), extract the best skills, agents, and commands, score them with a deterministic rubric + optional semantic self-scoring, and install a curated set into `~/.claude/` with full orchestration for project lifecycle guidance.

**Zero external cost.** No Anthropic API, no paid services. Uses existing Claude Code session for optional semantic scoring.

## Architecture — 9-Stage Pipeline

```
1. DISCOVER     scanner.sh              TSV inventory
2. EXTRACT      extractor.py            catalog.json (rich metadata)
3. SCORE        scorer_rubric.py        100-point rubric (+ self-scoring)
4. CURATE       curator.py              selected.json (domain grouped)
4.5 ORCHESTRATE orchestrator.py         lifecycle + /start-project
4.6 BUDGET      budget.py               token cost profile
4.7 VALIDATE    validate_agents.py      agent decision tree
5. INSTALL      installer.sh            ~/.claude/ deploy (staged + backup)
5.5 SMOKE TEST  smoke_test.sh           structural verification
6. OPTIMIZE     tracker.sh              usage tracking + pruning
```

## Running It

### First Time (from clone)
```bash
git clone <this repo>
cd CloaudeCodeCTO
bash scripts/bootstrap.sh
```

### Regular Use
```bash
# Check for updates, interactive
bash scripts/setup.sh

# Just inspect state
bash scripts/setup.sh --check

# Dry-run (no changes)
bash scripts/setup.sh --dry-run

# Auto mode (cron)
bash scripts/setup.sh --auto
```

## Key Design Decisions

- **No hard caps:** We don't force a max skill count or size limit — quality > quantity
- **Rubric + semantic scoring:** Deterministic rubric first, then optional semantic self-scoring for borderline cases
- **Domain classification:** Priority domains are project-mgmt → docs → testing → coding → architecture → devops
- **Interactive by default:** Every destructive action requires confirmation
- **Resumable:** Each pipeline stage writes to `decisions/`, can be re-run from any point
- **Factory-reset compatible:** Installer handles empty `~/.claude/` correctly
